CSS Variables

JET CSS variables are production in Redwood in JET 10.

Download and Run

Theme builder is basically just a mash up of the JET components, which allows you to more easily see the effect of a creating a theme. To start with we'll run theme builder with the default Redwood theme.
  1. Install the tooling. If you have used our tooling before you may want to uninstall and reinstall the CLI each time you upgrade to a new version of JET, here are the commands:
    • npm uninstall -g ojet-cli
    • npm install -g @oracle/ojet-cli
  2. Download JET-Theme-Builder-CSS-Vars.zip into a directory of your choice
  3. From a directory where you want to create a new project, run the following commands:
    1. ojet create JET-Theme-Builder-CSS-Vars --template=<path-to-downloaded .zip file>
    2. ojet add theming
    3. ojet serve

You should now see theme builder running with the default Redwood theme.

Now let's try running the purple demo theme provided with theme builder, the files for the purple theme are in the directory src/themes/purple. The CSS Variables documentation page includes information about creating a neutral and brand ramp. You can see these variable values in action by running ojet serve --themes=purple.

Create and Edit a Theme

Now let's create a new theme named 'xyz'. Note that 'xyz' is just an example name, use whatever name you'd prefer:
  • Run the following commands
    • ojet create theme xyz --basetheme=stable
      • This will create the xyz theme files under src/themes/xyz/
      • To extend Redwood instead of Stable use --basetheme=redwood instead, see the available themes doc for more information
    • ojet serve --themes=xyz
      • this will compile xyz and inject the generated CSS file in index.html
  • Open up src/themes/xyz/web/_xyz.cssvars.settings.scss in your favorite text editor and edit variables. As a test find and the line with --oj-html-font-size and change the value to 2em, which should cause everything to be big. Save the file, saving will cause the theme to automatically rebuild, after which you can view the changes in the browser.
  • More information about theming is available in the doc, demos, and dev guide.

Contrast Testing

To comply with WCAG 2.1 your theme needs an accessible contrast ratio for text and images as specified in WCAG 2.1 - Section 1.4.3 "Contrast". To test this we recommend the aXe chrome extension.
